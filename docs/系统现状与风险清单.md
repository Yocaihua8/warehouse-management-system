## 1. 当前项目判断

- 项目已形成完整业务闭环：登录鉴权 → 主数据 → 库存 → 手工入/出库 → AI 识别确认 → 打印/导出

- Web 端、桌面端、Python AI 服务并存，功能覆盖仍有差异

## 4. 当前剩余高风险项

### 4.1 端口冲突导致"误判为启动失败"

- 后端 `8080`、AI `9000` 都可能因已有进程占用而报启动失败（常见 `WinError 10048` / `Port 8080 already in use`）
- 这类问题是环境冲突，不是代码回归

### 4.2 配置来源仍不够收敛

- `application.yaml` / `application-local.yml` / `docker-compose.yml` 三套来源并存
- 本地手工启动与 Compose 启动默认值不同（会话存储、数据库密码）

### 4.3 Web 与桌面端能力仍有差异

- 桌面端部分功能在 Web 端尚无对应页面
- 不能默认视为同一产品面

---

## 5. 运行边界与排障建议

### 启动顺序

`MySQL → 后端(8080) → Python AI(9000，可选) → 前端(5173)`

### 快速健康检查

- 后端：`http://127.0.0.1:8080/system/health`
- AI：`http://127.0.0.1:9000/health`

### 端口冲突先查再启

```
netstat -ano | findstr :8080
netstat -ano | findstr :9000
```
