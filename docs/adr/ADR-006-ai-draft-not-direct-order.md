# ADR-006: AI 识别结果不直接生成正式单据

## 状态
已采纳（2025-08）

## 背景

AI OCR 识别的准确率受图片质量、商品名称规范性、OCR 噪声等因素影响，不能保证 100% 准确。需要决定：识别完成后是否直接生成正式入库/出库单，还是进入人工复核流程。

## 决策

AI 识别结果**必须经过人工复核确认**，不直接生成正式单据。

识别状态机：
```
pending（上传中）→ success（识别完成，等待确认）→ confirmed（人工确认，正式单生成）
                → failed（识别失败，终态）
```

Java 后端将识别结果保存为 `ai_recognition_record` + `ai_recognition_item` 草稿状态，不立即写库存。用户在前端复核/修正后，调用确认接口（Admin-only）才创建正式单据并更新库存。

识别记录与正式订单通过 `confirmed_order_id` 关联。

## 后果

**正面影响：**
- 规避 OCR 错误直接影响库存数据的风险
- 用户可在确认前修正识别结果（商品匹配、数量、单价）
- 未确认的识别记录不影响库存，可随时丢弃
- 审计链完整：识别记录 → 人工确认操作 → 正式单据

**负面影响 / 权衡：**
- 录单流程多一步（识别 → 人工确认），效率略低于全自动
- 需要维护"AI 草稿"状态机，实现复杂度增加
- AI 确认为 Admin-only，操作员需等待管理员确认

## 备选方案

**识别后直接生成草稿订单**：跳过独立的 AI 草稿状态，将识别结果直接映射为普通草稿订单，然后走正常确认流程。优点是流程更简，缺点是识别结果混入正式草稿池，难以区分"AI生成"和"手工录入"，已放弃。

**自动确认高置信度识别结果**：识别置信度超过阈值时自动确认。当前商品匹配准确率无法保证达到安全阈值，未来可作为增强功能评估。
