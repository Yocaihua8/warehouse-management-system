# AI 辅助开发约定（Codex Guide）

> 本文档约定使用 AI 辅助工具（Claude Code、Codex 等）参与本项目开发时的行为准则。

---

## 核心原则

- 仅做最小化、高置信度的改动
- 每一步改动后应用保持可运行状态
- 不重构无关模块
- 不改变 UI 外观（除非明确要求）
- 每次改动需清晰解释

---

## 推荐工作流

1. 先读取相关文件，理解上下文
2. 提出简短计划
3. 如涉及多文件改动，等待确认
4. 每次只实施一个小步骤
5. 提供验证方式

---

## 后端优先检查项

按以下顺序排查问题：
1. 请求方法映射（`@GetMapping` / `@PostMapping` 等）
2. URL 路径与 Controller 映射
3. DTO / Entity 字段名一致性
4. MyBatis Mapper 签名匹配
5. JSON 响应结构一致性

---

## 前端优先检查项

- 不重新设计页面
- 优先修复 API 调用、参数传递、渲染逻辑
- 尽量保持现有组件结构

---

## 安全规则

执行以下操作前**必须确认**：
- 删除文件
- 安装新依赖包
- 修改数据库配置
- 批量移动文件
- 修改构建配置
