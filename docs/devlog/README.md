# DevLog 开发日志

> 状态：Active  
> Owner：Yocaihua8  
> Last Updated：2026-04-18

用于记录**开发过程中的阶段进展、临时决策、阻塞问题与下一步动作**。  
这里保留过程事实，不承担定稿设计职责。

## 1. 定位与边界

| 文档 | 面向 | 内容 | 节奏 |
|------|------|------|------|
| [../../CHANGELOG.md](../../CHANGELOG.md) | 使用者 / 维护者 | 已发布或待发布的对外变更 | 按版本 |
| [../BACKLOG.md](../BACKLOG.md) | 团队 / 自己 | 待办、已知问题、技术债 | 随时更新 |
| [../adr/README.md](../adr/README.md) | 团队 | 重大/定稿的架构决策 | 按决策点 |
| `./*.md` | 开发者自己 | 今天/这轮做了什么、卡在哪、下一步 | 按天或按阶段 |

一句话区分：
- `CHANGELOG` 记录“对外有什么变化”
- `BACKLOG` 记录“还有什么要做”
- `ADR` 记录“为什么这么定”
- `DevLog` 记录“最近这轮做到哪里了”

## 2. 使用规则

- 只写过程性事实，不写大段定稿设计
- 临时决策如果影响多个模块，应提升到 `docs/adr/`
- 持续性的待办、问题和技术债，应同步沉淀到 [BACKLOG.md](../BACKLOG.md)
- 一条日志尽量控制在可快速浏览的范围内，优先写事实、阻塞、下一步

## 3. 文件命名

- 日报：`YYYY-MM-DD.md`
- 同一天有新增进展，继续追加到当天文件，不新建第二个文件
