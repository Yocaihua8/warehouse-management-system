# 开发任务拆解清单

## 1. 文档目的

本文档用于把“Java 仓库管理系统 + Python AI 辅助单据识别模块”拆解成可执行任务，避免开发过程中出现：

- 目标不清
- 并行过多
- 先后顺序混乱
- 写到一半返工

本文档遵循原则：

- 主线先闭环
- 辅助模块后接入
- 一次只推进一个明确阶段
- 每个阶段都有可验证产出

---

## 2. 总体阶段划分

整个项目建议分为 6 个阶段：

1. 项目前期设计完成
2. Java 前端闭环完成
3. AI 接口与数据库扩展完成
4. Python 识别服务原型完成
5. Java-Python 联调与草稿回填完成
6. 全链路测试与版本收口

每周只设 1 个主目标，不要并行太多。

## 11. 本阶段结论

当前最正确的推进方式不是同时重写一切，而是：

1. 先完成 Java 主系统前端闭环  
2. 再搭 Java 承接 AI 的骨架  
3. 再接 Python 服务原型  
4. 最后完成联调和收口
